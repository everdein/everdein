## Featured Work

### [Pay Period Planner](https://github.com/everdein/pay-period-planner)

An authenticated household cash-flow workspace for planning the next pay
period across income, recurring bills, savings, debt, and important dates.

I built it to solve a recurring planning problem, then evolved it from a local
JSON prototype into a PostgreSQL-backed, workspace-isolated application with
versioned aggregate saves and explicit architectural tradeoffs.

## How I Work

- Start with the user problem and make tradeoffs explicit.
- Keep state ownership, API contracts, and persistence boundaries understandable.
- Treat accessibility, failure states, concurrency, and security as product
  behavior.
- Use tests, automated checks, and documentation as engineering evidence.
- Prefer focused systems that can evolve over speculative complexity.
